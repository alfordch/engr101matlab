3.1. Warm Up¶

\[ESP = \frac{Na}{K + Ca + Mg + Ca}\]

Exercise: Function Basics¶

Which of these are correct implementations of a function square() that returns the square of a number (e.g. x * x)?

    Q4:

    function [ s ] = square( x )
  s = x .* x;
end
  • Correct
  • Yep, this one is correct! The variable s is computed by multiplying the input x by itself, and the result is returned since s is designated as the return variable.
  • Incorrect
  • This one is actually ok. The variable s is computed by multiplying the input x by itself, and the result is returned since s is designated as the return variable.

    Q5:

    function [ x ] = square( s )
  s = x .* x;
end
  • Correct
  • This one has an error. Check the input parameter and return variable...something's not quite right there.
  • Incorrect
  • You're right, this one has an error. In particular, the input and return variables are switched around in the function header. s should be the return variable (output) and x should be the input parameter.

    Q6:

    function [  ] = square( x )
  return = x .* x;
end
  • Correct
  • This one has an error. It has to do with the way MATLAB specifies what gets returned from a function.
  • Incorrect
  • You're right, this one has an error. Some programming languages use an explicit return statement, but in MATLAB you never have to type "return". Instead, the function should specify what variables to return inside of the square brackets [ ].

    Q7:

    function [  ] = square( x )
  return = x .* x;
end
  • Correct
  • This one has an error. It has to do with the syntax in the function header.
  • Incorrect
  • You're right, this one has an error. The return variable must be specified with square brackets [ ] rather than parentheses ( ). MATLAB and other programming languages are pretty strict about syntax, but the good news is MATLAB will generally underline syntax errors as you're writing the code so that you can fix it.

    Q8:

    function [ x ] = square( x )
  x = x .* x;
end
  • Correct
  • Yep, this one is correct! It is fine to use the same variable for input and to return (for output). x starts with the input value, and is then updated inside the function before its new value is returned.
  • Incorrect
  • This one is actually ok. It is fine to use the same variable for input and to return (for output). x starts with the input value, and is then updated inside the function before its new value is returned.

    Q9:

    function [  ] = square( x )
  return = x .* x;
end
  • Correct
  • This one has a sneaky mistake. It actually computes and returns the right value, but it does not follow best design practices for functions... See if you can identify what's wrong on your own, or select "incorrect" and check the answer to see an explanation.
  • Incorrect
  • Although this implementation computes the right answer, it doesn't suppress output for individual computations. We don't want noisy functions, so each of the assignments in the implementation should end with a semicolon ; to suppress the output.

The nutrient cycle of Proxima B is not yet fully understood, so our scientists want to check whether the ESP changes over time. To do this, we’ll use a dataset with soil samples taken at the same location every day throughout a year.

For simplicitly, we will assume all months have 30 days, such that there are 360 samples total. The data are stored in a file called daily_samples.csv.

In this exercise, you’ll read in that data using the csvread() function and compute the ESP for all daily samples.

Create a new script called AnalyzeESP.m and write code to:

Step 1

Read the data from daily_samples.csv into a variable called dailySamples using the csvread() function. Make sure to skip the first row, which contains the column headings.

Step 2

Set Na, K, Ca, and Mg variables to hold the data from each corresponding column in the dailySamples matrix.

Hint

Use row/column indexing to select each column from dailySamples. For example, to find the sodium values, you would use Na = dailySamples(:,2) to assign the first column data into Na. (If you skipped the first column using csvread() above, it would be Na = dailySamples(:,2) instead.)

Step 3

Use the ESP() function to compute a vector of ESP values for each of the samples (you should pass in the variables from the previous step). Store the result in a variable called esp.

Note

This kind of script in general is called a “driver” (sometimes “driver script” or “driver program”. Why? It’s basically the script that controls and brings together functionality from other parts of the program (e.g. reading in data files or calling functions) to accomplish some task, just like the driver of a car controls the steering wheel, pedals, etc. to get where they’re going.

Exercise: Monthly Average of ESP¶

In this exercise, you’ll work with the ESP data from your write a function to compute the average ESP value for a given month.

Exercise Files

This exercise uses the same daily_samples.csv and ESP.m files as the previous one. See above for the download links.

Create a new function file called monthlyAverage.m.

In that file, define the function monthlyAverage(). It takes in a vector of 360 ESP values and a month number (e.g. 1 is January, 12 is December) and returns the average of ESP values for samples during the given month. For example, given the esp vector, you could call monthlyAverage(esp, 3) to compute the average ESP during March (the 3rd month).

Step 1

To find the range of days to average based on the month, use these formulas:

  • startDay = 30*(month-1) + 1;

  • endDay = startDay + 29;

Step 2

To compute the average, use MATLAB’s built-in mean() function on a subvector of the ESP values selected according to the range of days you just computed.

Hint

  • Use indexing and range notation to select the subvector of values to average. For example, to select values for January, you could write data(1:30). Of course, you don’t want to hardcode 1:30 - instead use your variables containing the appropriate startDay and endDay.
